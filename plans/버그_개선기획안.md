## 🐛 발견된 버그

### 1. `_excel_color_to_hex()` — 검정색 누락 버그 ⚠️ 기획안 오류 수정
**파일**: [`ppt_extractor_v3.py`](../ppt_extractor_v3.py) (라인 838-849)

**기존 기획안 판단**: "BGR→RGB 변환 로직 오류" → ❌ **틀림**

**실제 코드 검증**:
```python
b = (color >> 16) & 0xFF  # Blue 추출 ← 올바름
g = (color >> 8)  & 0xFF  # Green 추출 ← 올바름
r = color & 0xFF          # Red 추출 ← 올바름
return f"{r:02X}{g:02X}{b:02X}"  # RGB 순서 반환 ← 올바름
```
→ BGR 추출 후 RGB 순서로 반환하는 로직은 **정상 동작함**

**진짜 버그**: `color == 0` 조건에서 `None` 반환
```python
if color is None or color == 0:  # ← color=0은 순수 검정(#000000)
    return None                  # ← 검정색이 항상 무시됨!
```
- RGB(0,0,0) = 검정색이지만 `0`으로 평가되어 색상 없음으로 처리됨
- 검정색 텍스트·배경이 전부 투명/기본값으로 추출되는 실제 렌더링 오류

**해결 방향**: `color is None`과 `color == 0` 조건 분리 필요

### 3. `_print_excel()` PrintOut fallback 무동작 버그
**파일**: [`pdf_printer.py`](../pdf_printer.py) (라인 335-359)

**문제**: `ExportAsFixedFormat` 실패 시 fallback으로 `PrintOut`을 사용하지만, `PrintOut`은 PDF를 직접 생성하지 못함.

```python
except:
    workbook.PrintOut(
        PrintToFile=True,
        PrToFileName=save_path,  # ← PrintOut은 PDF 저장 불가 (XPS/PRN 형식)
        Copies=1
    )
```

- `PrintOut(PrintToFile=True)`는 프린터 드라이버로 출력을 파일에 저장하는 방식
- PDF 저장은 PDF 프린터 드라이버가 설치된 경우에만 우연히 동작
- `ExportAsFixedFormat`이 실패하는 환경에서 `PrintOut`도 사실상 실패함
- **실질적으로 fallback이 없는 것과 동일**

**해결 방향**: fallback을 제거하거나, win32print 활용 또는 오류 메시지 안내로 대체

### 4. Bare `except:` 구문 남용
**파일**: 전체 프로젝트 — [`ppt_extractor_v3.py`](../ppt_extractor_v3.py) 집중

**문제**: `_handle_image_shape()` 단일 메서드 내에만 bare `except:` 4개 존재

```python
# ppt_extractor_v3.py 라인 979, 990, 1002, 1012
except:          # ← KeyboardInterrupt, SystemExit까지 삼킴
    return False # ← 어떤 예외인지 로깅조차 없음
```

- 어떤 예외가 발생했는지 추적 불가 → 디버깅 불가능
- `KeyboardInterrupt` / `SystemExit` 같은 심각한 시그널도 무시됨
- 로거(`self.logger`)가 있음에도 예외를 기록하지 않음

**해결 방향**: `except Exception as e: self.logger.error(...)` 로 변경
